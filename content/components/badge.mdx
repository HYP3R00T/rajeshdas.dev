## Overview

The Badge component is a versatile label element designed to highlight status, categories, or metadata. It provides multiple visual styles and is perfect for tagging, labeling, and displaying compact information.

## Usage

```astro
---
import Badge from "@/components/ui/Badge.astro";
---

<Badge variant="default">New</Badge>
```

## Variants

The badge comes in five different visual styles:

<Badge variant="default">Default</Badge>
<Badge variant="secondary">Secondary</Badge>
<Badge variant="destructive">Destructive</Badge>
<Badge variant="outline">Outline</Badge>
<Badge variant="accent">Accent</Badge>

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"default" \| "secondary" \| "destructive" \| "outline" \| "accent"` | `"default"` | The visual style variant |
| `class` | `string` | `undefined` | Additional CSS classes |

## Design Tokens

The Badge component uses the following design system colors:

- **Default**: Light text on elevated background with border
- **Secondary**: Default text on elevated background with border
- **Destructive**: Default text on red background
- **Outline**: Default text with border, transparent background
- **Accent**: Accent text on translucent accent background (accent/20)

## Accessibility

- The component uses semantic `<span>` elements
- Text is automatically uppercase with wide tracking for readability
- Focus states are handled with visible ring indicators
- Maintains sufficient color contrast for all variants
- Can be combined with screen reader text for additional context

## Best Practices

- Use badges sparingly to maintain visual hierarchy
- Choose variants that match the semantic meaning (e.g., destructive for errors)
- Keep badge text concise (1-2 words ideal)
- Use the accent variant to highlight important or featured items
- Combine with other components for richer UI patterns
