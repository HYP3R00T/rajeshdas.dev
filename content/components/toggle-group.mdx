## Overview

The ToggleGroup component displays a set of buttons where only one can be selected at a time. It's perfect for filtering, toggling between views, or selecting a single option from a group.

## Usage

```astro
---
import ToggleGroup from "@/components/ui/ToggleGroup.astro";
---

<ToggleGroup
  name="view"
  value="grid"
  options={[
    { label: "Grid", value: "grid" },
    { label: "List", value: "list" },
  ]}
/>
```

## Filtering Example

For filtering blog posts or projects by category:

```astro
---
import ToggleGroup from "@/components/ui/ToggleGroup.astro";
---

<ToggleGroup
  name="category"
  value="all"
  options={[
    { label: "All", value: "all" },
    { label: "DevOps", value: "devops" },
    { label: "Cloud", value: "cloud" },
    { label: "Infrastructure", value: "infrastructure" },
  ]}
/>
```

## Horizontal (Default)

Buttons arrange left-to-right:

```astro
<ToggleGroup
  name="difficulty"
  value="intermediate"
  orientation="horizontal"
  options={[
    { label: "Beginner", value: "beginner" },
    { label: "Intermediate", value: "intermediate" },
    { label: "Advanced", value: "advanced" },
  ]}
/>
```

## Vertical

Buttons stack vertically:

```astro
<ToggleGroup
  name="difficulty"
  value="intermediate"
  orientation="vertical"
  options={[
    { label: "Beginner", value: "beginner" },
    { label: "Intermediate", value: "intermediate" },
    { label: "Advanced", value: "advanced" },
  ]}
/>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `name` | `string` | Required | Form field name |
| `options` | `Array<{label: string; value: string}>` | Required | Available options |
| `value` | `string` | `undefined` | Currently selected option value |
| `orientation` | `"horizontal" \| "vertical"` | `"horizontal"` | Button layout direction |
| `class` | `string` | `undefined` | Additional CSS classes |

## Option Structure

Each option is an object with:

```ts
{
  label: string;  // Display text
  value: string;  // Form value when selected
}
```

## Styling

- Selected button: White background with black text (`bg-foreground-0 text-background-0`)
- Unselected button: Border only with hover effects
- Maintains consistent sizing and spacing with other form elements

## Client-Side Interaction

The component emits a custom `toggle-change` event when selection changes:

```astro
<script>
  window.addEventListener('toggle-change', (event) => {
    console.log('Selected:', event.detail.name, event.detail.value);
  });
</script>
```

## Accessibility

- Uses native HTML radio inputs (hidden)
- Full keyboard navigation support
- Proper ARIA attributes for screen readers
- Each option is independently focusable
- Clear visual indication of selected state
- Submit/filter results on selection via form or custom event handler
