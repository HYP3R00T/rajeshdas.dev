## About

This repository powers my personal website, where I publish technical writing, learning paths, notes, and selected project work.

The site is public because the work is public. It reflects how I think about systems, documentation, tooling, automation, and building things that stay useful over time.

## What You’ll Find

- Articles with long-form technical writing and implementation notes
- Notes for smaller ideas, observations, and topic-based fragments
- Learning content organized into skill maps and modules
- A homepage that highlights positioning, featured work, and current signals
- Project links that connect this site to the broader work published under `hyperoot.dev`

## Stack

### Frontend and Application Layer

- [Astro](https://astro.build/) for site architecture, routing, and static generation
- [React](https://react.dev/) for interactive islands
- [TypeScript](https://www.typescriptlang.org/) for typed components and shared utilities
- [Tailwind CSS](https://tailwindcss.com/) for styling

### UI and Design Tooling

- [shadcn](https://ui.shadcn.com/) for reusable UI composition
- [Radix UI](https://www.radix-ui.com/) for accessible primitives
- [class-variance-authority](https://cva.style/) for component variants
- [clsx](https://github.com/lukeed/clsx) for conditional class names
- [tailwind-merge](https://github.com/dcastil/tailwind-merge) for Tailwind class merging
- [tw-animate-css](https://www.npmjs.com/package/tw-animate-css) for animation helpers
- [Fontsource](https://fontsource.org/) for self-hosted typography

### Content and Publishing

- [MDX](https://mdxjs.com/) for rich content authoring
- [Astro's content collections](https://docs.astro.build/en/guides/content-collections/) for structured posts, notes, and learning content
- [RSS](https://docs.astro.build/en/guides/rss/) for syndication
- [Shiki](https://shiki.style/) for syntax highlighting
- [rehype-pretty-code](https://rehype-pretty.pages.dev/) for code block rendering

### Media and Asset Pipeline

- [astro-icon](https://www.astroicon.dev/) for Astro-native icons
- [Lucide](https://lucide.dev/) for the icon set
- [Plyr](https://plyr.io/) for media playback
- [Sharp](https://sharp.pixelplumbing.com/) for image optimization

### Development Environment and Runtime Management

- [Dev Containers](https://containers.dev/) for reproducible development environments
- [Docker](https://www.docker.com/) for the containerized development base
- [mise](https://mise.jdx.dev/) for runtime and tool version management
- [pnpm](https://pnpm.io/) for package management
- [astro-auto-import](https://www.npmjs.com/package/astro-auto-import) for authoring ergonomics
- [@astrojs/ts-plugin](https://docs.astro.build/en/guides/typescript/) for TypeScript integration

### Quality, Automation, and Repository Operations

- [Biome](https://biomejs.dev/) for formatting and linting
- [GitHub Actions](https://github.com/features/actions) for pull request quality checks
- [Dependabot](https://github.com/dependabot) for dependency maintenance
- [Commitizen](https://commitizen-tools.github.io/commitizen/) for commit message conventions
- [prek](https://prek.j178.dev/) for hook management
- [pre-commit](https://pre-commit.com/) for repository checks
- [rumdl](https://github.com/rvben/rumdl) for Markdown quality checks and formatting

### Editor and Authoring Experience

- [VS Code Dev Containers](https://code.visualstudio.com/docs/devcontainers/containers) for a curated editor setup with Astro, Tailwind, YAML, MDX, GitHub Actions, `mise`, and `rumdl` support

## Why It’s Open

This is a personal website, not a starter kit, but I keep the repository open so the implementation, writing structure, and design decisions stay visible for anyone curious about how the site is built or how the work is organized in public.
